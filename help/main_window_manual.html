  grade INTEGER,
  birth_date TEXT

üîß Troubleshooting

Common Issues

Symptom Possible Cause Solution
Buttons are disabled/grayed out No database is open Open or create a database first
"Cannot open database" error File is corrupted or in use Check if file exists, try backup restore
Search not working Wrong column/operation combination Check data type matches operation
Data not saving Database file is read-only Check file permissions, close other programs
Application crashes on close Unsaved changes conflict Always save before closing
Slow performance Large database file Use search filters, clear unused data

Error Messages Explained

"Failed to open database"
The database file might be:
1. In use by another program
2. Corrupted or incomplete
3. Missing file permissions
Solution: Close other programs, check file exists, restore from backup
"All database files will be deleted"
This appears when overwriting an existing database. The system will delete:
1. Main .db file
2. Journal files (.db-journal, .db-wal, .db-shm)
Note: This is normal for SQLite file cleanup
"Save changes before closing?"
You have unsaved changes. Options:
1. Save: Save changes and close
2. Discard: Lose changes, restore from backup
3. Cancel: Return to application
Tip: Use File ‚Üí Save regularly to avoid this

Recovery Procedures

1

Lost Data Recovery: Check for .backup files in the same folder as your database

2

Corrupted Database: Try opening the backup file instead of main .db file

3

Accidental Deletion: If you cleared data by mistake, immediately close WITHOUT saving and restore from backup

üí° Best Practice: Always create manual backups (File ‚Üí Get Backup) before making major changes or bulk deletions.
